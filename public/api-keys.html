Understanding Authentication: Account Token vs API Key
Account Token (phone_id + token)
  • What: Automatically generated when you create a WhatsApp connection
  • Purpose: Identifies and authenticates a specific WhatsApp phone number/account
  • Scope: One token per WhatsApp account
  • Where to get: Created at /multi-account.html when scanning QR code
  • Use for: Multi-Account API (/api/v2/*)
  • Headers: phone_id and token
  • Permissions: Full access to that specific WhatsApp account
  • Example: If you have 5 WhatsApp numbers, you'll have 5 different tokens (one per number)
API Key (X-API-Key)
  • What: Manually created security key for external applications
  • Purpose: Controls access to the Legacy API for third-party apps/services
  • Scope: Works with the default/primary WhatsApp account only
  • Where to get: Create below on THIS page
  • Use for: Legacy API (/api/external/*)
  • Headers: X-API-Key
  • Permissions: Customizable (send message, media, groups, etc.)
  • Example: You can create multiple API keys with different permissions for different apps
Key Differences:
Feature Account Token (phone_id + token) API Key (X-API-Key)
Generation Automatic (when connecting WhatsApp) Manual (create on this page)
WhatsApp Accounts Supports multiple accounts (1 token per account) Single default account only
Permissions Full access to the specific WhatsApp account Granular permissions (customizable)
Rate Limiting Per account basis Configurable per API key
IP Whitelist Not available Supported
Expiration Never expires (until account disconnected) Optional expiration date
Use Case Managing multiple WhatsApp business numbers Single account with controlled access for external apps
API Architecture Overview
Legacy API (Single Account)
  • Authentication: API Key based
  • Endpoints: /api/external/*
  • Use Case: Single WhatsApp connection
  • Header: X-API-Key: your_api_key
  • Get API Key: Create below using "Create New API Key" button
Multi-Account API
  • Authentication: Phone ID + Token based
  • Endpoints: /api/v2/*, /api/accounts/*
  • Use Case: Multiple WhatsApp connections
  • Headers: phone_id and token
  • Get Credentials: Visit /multi-account.html to create accounts
Which Should I Use?
Use Account Token (phone_id + token) if:
  • ‚úì You need to manage multiple WhatsApp numbers
  • ‚úì You want separate control for each phone number
  • ‚úì You're building a multi-tenant WhatsApp system
  • ‚úì Example: Agency managing 10 different client WhatsApp numbers
Use API Key (X-API-Key) if:
  • ‚úì You only have ONE WhatsApp number to manage
  • ‚úì You want granular permission control
  • ‚úì You need IP whitelisting for security
  • ‚úì You want to set expiration dates for access
  • ‚úì Example: CRM system connecting to your single business WhatsApp
Quick Summary:
  • Account Token = Auto-generated when you connect a WhatsApp ‚Üí Identifies WHICH WhatsApp account
  • API Key = Manually created on THIS page ‚Üí Controls WHO can access the default WhatsApp account
  • Both are credentials, but serve different purposes in different architectures

API Documentation & Examples

Legacy API Usage (API Key)

Use the API keys created below. Replace YOUR_API_KEY_HERE with your actual key.

curl -X POST http://localhost:3000/api/external/send-message \ -H "X-API-Key: YOUR_API_KEY_HERE" \ -H "Content-Type: application/json" \ -d '{"to": "1234567890", "message": "Hello!"}'
Create your API key below using "Create New API Key" button
Multi-Account API (Phone ID + Token)

Get phone_id and token from /multi-account.html page.

curl -X POST http://localhost:3000/api/v2/send-message \ -H "phone_id: your_phone_id" \ -H "token: your_account_token" \ -H "Content-Type: application/json" \ -d '{"to": "1234567890", "message": "Hello!"}'
Visit /multi-account.html to create multi-account connections
How to Get Your Credentials:
For Legacy API (X-API-Key):
  1. Click "Create New API Key" button below
  2. Fill in the details and create
  3. Copy the generated API key (shown only once)
  4. Use it in X-API-Key header
For Multi-Account API (phone_id + token):
  1. Go to /multi-account.html
  2. Click "Add New Account" and scan QR code
  3. After connecting, view account details
  4. Copy both phone_id and token
  5. Use them as separate headers in API calls

Send Message - Legacy API (PowerShell)
$body = @{
  to = "919876543210"
  message = "Hello from API!"
} | ConvertTo-Json

Invoke-WebRequest `
  -Uri "http://localhost:3000/api/external/send-message" `

import requests

headers = {

Response Format

All endpoints return JSON responses with the following structure:

Success:
{
  "success": true,
  "message": "Message sent successfully",
  "data": { ... }
}

Error:
{
  "success": false,
  "error": "Error description",
  "message": "Detailed error message"
}
Rate Limiting

Each API key has a configurable rate limit (default: 100 requests/minute). Exceeded limits will return:

{
  "success": false,
  "error": "Rate limit exceeded",
  "message": "Too many requests"
}
