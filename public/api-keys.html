API Key Management

Manage API keys for Legacy and Multi-Account architectures

API Architecture Overview
Legacy API (Single Account)
  • Uses API Keys for authentication
  • Endpoints: /api/external/*
  • Single WhatsApp connection
  • Header: X-API-Key: your_key
Multi-Account API
  • Uses Phone ID + Token authentication
  • Endpoints: /api/v2/*, /api/accounts/*
  • Multiple WhatsApp connections
  • Headers: phone_id + token

API Documentation

Legacy API Usage
curl -X POST http://localhost:3000/api/external/send-message \ -H "X-API-Key: YOUR_API_KEY_HERE" \ -H "Content-Type: application/json" \ -d '{"to": "1234567890", "message": "Hello!"}'
Multi-Account API Usage
curl -X POST http://localhost:3000/api/v2/send-message \ -H "phone_id: your_phone_id" \ -H "token: your_account_token" \ -H "Content-Type: application/json" \ -d '{"to": "1234567890", "message": "Hello!"}'

Example: Send Message (PowerShell)

$body = @{
  to = "919876543210"
  message = "Hello from API!"
} | ConvertTo-Json

Invoke-WebRequest -Uri "http://localhost:3000/api/external/send-message" `
  -Method POST `
  -Headers @{
    "X-API-Key" = "YOUR_API_KEY_HERE"
    "Content-Type" = "application/json"
  } `
  -Body $body

Example: Send Media (Python)

import requests

headers = {
  "X-API-Key": "YOUR_API_KEY_HERE",
  "Content-Type": "application/json"
}

data = {
  "to": "919876543210",
  "media": "https://example.com/image.jpg",
  "caption": "Check this out!"
}

response = requests.post(
  "http://localhost:3000/api/external/send-media",
  headers=headers,
  json=data
)
print(response.json())

Example: Send Location (Node.js)

const axios = require('axios');

const response = await axios.post(
  'http://localhost:3000/api/external/send-location',
  {
    to: '919876543210',
    latitude: 37.7749,
    longitude: -122.4194,
    description: 'San Francisco, CA'
  },
  {
    headers: {
      'X-API-Key': 'YOUR_API_KEY_HERE',
      'Content-Type': 'application/json'
    }
  }
);
console.log(response.data);

Response Format

All endpoints return JSON responses with the following structure:

Success:
{
  "success": true,
  "message": "Message sent successfully",
  "data": { ... }
}

Error:
{
  "success": false,
  "error": "Error description",
  "message": "Detailed error message"
}

Rate Limiting

Each API key has a configurable rate limit (default: 100 requests/minute). Exceeded limits will return:

{
  "success": false,
  "error": "Rate limit exceeded",
  "message": "Too many requests"
}
